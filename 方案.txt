灵活用工平台方案（承揽后分包模式）

〇、 模式定位与法律关系（本方案前提，所有模块围绕此展开）

本平台采用"承揽后分包"模式，与撮合/居间模式的本质区别：

  - 法律关系链：用工企业(B) ↔ 平台 签订《总承揽合同》，平台 ↔ 零工(C) 签订《分包协议》。
    平台是交易主体和责任主体，不是居间方。
  - 开票方向：平台向 B 端全额开具服务发票（6%现代服务或对应品目），B 端凭此发票即可
    完成企业所得税税前扣除（无需复杂凭证包）。
  - 风险转移：业务真实性风险、虚开发票刑事风险、C 端人身伤害的发包方责任、
    平台自身成本的税前扣除举证责任，全部集中于平台自身。
  - 因此本方案的核心命题是：对 B 简化（一张发票解决扣除），对内强化（平台自留
    完整证据链 + 税负管理 + 风险隔离）。

一、 核心功能架构蓝图

平台按业务流向和使用主体，划分为四端：

1.  企业端（B端）：任务发布、签订总承揽合同、资金充值（银行存管户）、验收结算、接收全额发票。
2.  零工端（C端）：实名认证、签订分包协议、接单交付、收入提现、税务查询与年度扣缴凭证下载。
3.  平台运营端（Admin）：业务审核、行业准入管理、风控预警、税负测算、税务配置、涉税信息报送、自留证据链归档。
4.  外部接口端（API）：对接公安（实名）、存管银行（虚拟账户分账）、税务局（申报与报送）、数电票开票服务、电子签、保险公司（按单投保）。

二、 详细功能模块设计

1. 认证与档案管理模块（满足"实名核验"与"报送基础"要求）

  - C端（零工）实名核验系统
      - 身份认证：身份证OCR、活体人脸识别比对（强制要求，确保业务真实性）。
      - 收款账户绑定：本人一类银行卡绑定（必须三要素/四要素一致，防洗钱）。
      - 主体属性标签：自然人 / 个体工商户标记。
        ※ 此标签为算税引擎的路由开关（见模块4），录入个体户需上传营业执照并核验经营范围与开票能力。
      - 设备指纹与IP画像采集：注册即采集，用于防批量注册养号（见模块8）。
  - B端（用工企业）资质核验系统
      - 企业营业执照OCR、法人认证、对公账户绑定。
      - 行业准入校验：比对行业负面清单（建筑劳务分包、医美导流、直播打赏、
        贸易走账类等高风险行业拒绝准入或人工加强审核），准入结论留痕。
      - 企业发票抬头与税收分类编码配置。

2. 电子合同与签约模块（承揽模式法律链条的载体，新增）

  - 双层合同体系
      - B端入驻签订《总承揽框架合同》，每笔任务生成《任务工单》作为框架合同附件，
        形成"框架合同 + 工单"两级结构，避免每单重签。
      - C端首次接单前签订《分包协议（框架）》，每次接单生成确认工单。
  - 承揽特征条款固化（对抗"事实劳动关系"认定的核心）
      - 分包协议模板强制包含：按成果/计件计酬、不约定固定工作时间、工具设备自备、
        不接受用工方考勤管理、成果不合格不计酬等条款。
      - 任务发布模型联动限制：禁止发布"按月固定薪资""按日打卡计酬"类任务，
        计酬方式字段只允许 按成果/按件/按单（与模块8伪劳务风控联动）。
  - 对接第三方电子签（CA认证、时间戳、司法存证），全部合同与工单在线签署、在线归档。

3. 任务与交易管理模块（满足"业务交易明细真实性"要求）

  - 任务大厅与撮合
      - 企业发布任务（需包含：任务名称、交付标准、预算、周期、计酬方式、分类如设计/翻译/技术等）。
      - 零工接单/报名，企业确认录用，系统自动生成分包工单并完成电子签。
  - 在线交付与验收（证据链核心）
      - 交付物上传：文档、代码仓库链接、设计图、工作记录等留痕。
      - 验收确认流：企业端点击验收合格，生成带时间戳的《业务交易确认单》，
        并自动触发开票流程（见模块7）与结算流程（见模块5）。

4. 智能税务计算引擎（核心竞争力：落实16号公告，按主体类型分支路由）

  - 入口路由：按 C 端主体属性标签分两条完全不同的处理线
      【A线：自然人 —— 劳务报酬所得，平台为法定扣缴义务人】
      - 个税"累计预扣法"计算器
          - 连续用工月份追踪：系统自动记录该C端用户在本平台的"当年连续取得报酬月份数"。
          - 个税自动算税公式配置：(当月及此前总收入 * (1 - 20%) - 5000元/月 * 连续月份数)
            * 七级累进税率 - 速算扣除数 - 已预扣税额。
      - 增值税代办计算器
          - 免税额度监控：自动按月统计C端月销售额，判断是否低于10万元（免征增值税）。
          - 优惠税率应用：超额部分按政策自动适用1%或对应减按征收率计算增值税及附加税。
      【B线：个体工商户 —— 经营所得，平台不代扣个税】
      - 平台不扣缴，结算时校验其开票能力，要求其向平台开具发票（数电普票/专票）后付款，
        发票自动与工单、支付流水关联归档（构成平台进项与成本凭证）。
      - 提示其自行办理经营所得申报，平台提供收入明细导出。
  - 市场主体登记引导（双阈值）
      - 引导阈值（新增，下调至现实区间）：自然人C端月收入持续超过10万元，
        系统推送个体户注册引导（注册后转入B线，平台获得进项改善税负，C端享受经营所得核定/查账政策）。
      - 强制阈值：滚动12个月累计收入达到或接近500万元时，锁定其自然人接单权限，
        强制走"市场主体工商登记引导"流程。
  - 年度扣缴凭证：每年初为A线C端生成上年度《个人所得税扣缴凭证》供下载，
    支撑其跨平台收入的个人汇算清缴。

5. 资金存管与清结算模块（满足"结算支付记录"要求，规避二清）

  - 银行存管前置（新增，硬性约束）
      - 接入持牌银行虚拟账户分账体系（如平安见证宝、网商银行等同类产品），
        B端充值直接进入银行存管虚拟户，平台只发分账指令、自身不触碰在途资金。
  - 企业资金户管理：B端存管户充值、冻结（发布任务时锁定资金）、余额提醒。
  - 清结算分账系统：任务验收后，存管银行按指令自动拆分：
      - 应发分包款 -> C端银行卡
      - 代扣个税、代办增值税及附加税 -> 平台税务备付金账户
      - 平台毛利（承揽价 - 分包价 - 税费）-> 平台收益账户
  - 支付流水生成器：生成含交易单号、付款方、收款方、明细金额的《结算支付流水凭证》。
  - 自动对账模块（新增）：银行存管流水与系统订单流水T+1自动对账，
    差异自动挂起并告警，对账结果留痕归档。

6. 平台税负测算与报价管理模块（承揽模式特有，新增）

  - 背景：平台销项为全额6%，自然人C端无法提供进项，存在增值税税负倒挂风险。
  - 按单税负测算器：承接任务前自动测算 毛利 =承揽价 - 分包价 - 增值税及附加（销项-可得进项）
    - 代扣税费成本 - 保险成本 - 通道费，低于安全线的承揽单自动预警或拒接。
  - 进项优化看板：统计个体户C端占比、可得进项规模，量化"引导C端注册个体户"
    对平台税负的改善效果，反哺模块4的引导策略。
  - 月度税负健康度报表：增值税实际税负率、毛利率、各行业线盈亏，供经营决策。

7. 发票与凭证档案中心（双重定位：对B简化、对内自证）

  - 销项：对B端全额开票（B端的税前扣除凭证）
      - 对接数电票开票服务（乐企或服务商接口），验收确认即自动全额开具服务发票，
        发票与《业务交易确认单》《任务工单》一一关联（本身即防虚开证据链一环）。
  - 平台自留证据链（承揽模式的生命线：支撑平台自身成本扣除 + 对抗虚开认定）
      - 按单归集"四流"档案：合同流（总承揽合同+工单+分包协议）、业务流（交付物+验收单）、
        资金流（存管支付流水）、发票流（销项发票 + B线C端进项发票 / A线代扣完税凭证）。
      - A线自然人成本扣除凭证包：分包协议 + 交付验收记录 + 支付流水 + 个税代扣完税凭证，
        四件套自动按单装订，支撑平台企业所得税税前扣除。
  - B端业务真实性证明包（简化版，按需提供）
      - 《实名核验记录证明》《业务交易明细表》《资金结算支付水单》，
        供B端应对税务核查时自证业务真实（非扣除必需）。
  - 数据电子存证
      - 所有相关交易与操作日志通过区块链或可信云固化，保存期限满足税法规定（一般为10年）。

8. 风控与防伪预警系统（承揽模式下风险集中于平台，全面加强）

  - 劳动关系隔离风控
      - 防"员转零"监测：比对B端过往发薪名单与当前零工名单，若发现企业将大量原全职
        员工转至本平台接单，触发高风险预警并阻断。
      - 工时/频次异常监控：识别"每天固定打卡、按月固定金额"等伪劳务特征，
        与模块2的计酬方式限制双重拦截，平台拒绝承接。
  - 交易真实性风控（防B端借平台虚构业务走账、虚开发票——承揽模式下刑责直接落在平台）
      - 四流合一系统级硬校验：无签约工单不可结算、无交付物不可验收、
        无验收不可开票、无开票依据不可分账（事前阻断，而非事后审计）。
      - 集中度监控：单C端对单B端收入集中度、单C端月收入限额、同设备/同IP多账号
        关联图谱，超阈值冻结并人工审核。
      - B端业务抽查回访：按比例对已结算任务电话/在线回访C端，确认真实提供服务，回访记录留痕。
      - 行业负面清单动态维护：高风险行业准入拦截（与模块1联动）。
  - 反洗钱：大额/高频/夜间异常交易监测，可疑交易报告流程。

9. 保险保障模块（发包方责任隔离，新增，一票否决级）

  - 按单自动投保：C端接单即自动投保意外险/雇主责任险，接单生效、交付终止，
    保费计入平台按单成本（纳入模块6税负测算）。
  - 任务类别分级：涉线下作业（配送、安装、施工辅助等）强制投保高保额方案；
    纯线上任务（设计、翻译、开发）可选基础方案。
  - 理赔通道：C端App内一键报案，平台协助理赔，全程留痕（证明平台已尽安全保障义务）。

10. 涉税信息报送与代办模块（落实15号公告与涉税报送规定）

  - 平台初始报送模块：平台上线从事互联网经营30日内，一键导出/API直连报送平台基本信息（向税务机关）。
  - 季度例行报送系统
      - 按季度自动打包生成《平台内从业人员身份信息及收入信息明细表》。
      - 支持税务局要求的XML/Excel格式导出，或税务系统API直连接口申报。
  - 税款代扣代缴/代办申报工作台
      - 汇总全平台当月代扣的个税（A线）、代办的增值税，生成申报表，
        供平台财务向税务系统进行批量申报缴款。

三、 系统核心流程（Happy Path，承揽后分包版）

1.  入驻：零工(C)完成活体实名并签订《分包协议（框架）》-> 企业(B)通过行业准入审核、
    签订《总承揽框架合同》并向银行存管户充值。
2.  承接：企业发单（系统校验计酬方式合规）-> 平台税负测算通过 -> 生成任务工单。
3.  履约：零工接单（生成分包工单+电子签+自动投保）-> 零工上传交付物 -> 企业验收确认，
    生成《业务交易确认单》。
4.  算税：引擎按主体标签路由——自然人走累计预扣个税+增值税阈值判定；
    个体户校验进项发票后免扣缴。
5.  结算：系统向存管银行发分账指令 -> 分包款打至零工账户、税金入备付金户、
    毛利入平台收益户 -> T+1自动对账。
6.  开票：验收触发数电票自动全额开具 -> 推送B端（即B端税前扣除凭证）。
7.  报送：月底平台统一代办增值税和代扣个税申报缴款 -> 季度初向税务机关报送
    上季度从业人员身份与收入信息。
8.  归档：按单归集四流证据链（平台自留）+ B端业务真实性证明包（按需）->
    区块链/可信云存证10年；年度初生成C端个税扣缴凭证。

四、 建设优先级建议

  - P0（不上线不可经营）：实名认证、电子合同双层签约、银行存管分账、
    算税引擎（含主体路由）、数电票对接、四流硬校验。
  - P1（首月内必须具备）：按单保险、涉税报送、自动对账、平台自留证据链归档。
  - P2（运营优化）：税负测算与报价管理、个体户引导、集中度风控图谱、抽查回访工作台。
