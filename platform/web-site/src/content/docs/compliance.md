## 什么是「四流合一」

合规的核心，是让**合同流、业务流、资金流、发票流**四者相互印证、指向同一笔真实交易：

- **合同流**：总承揽框架合同 → 分包协议 → 任务工单 → 分包工单，全节点签署时**正文快照存档**。
- **业务流**：发布 → 报名 / 派单 → 录用 → 交付 → 验收，每一步留痕。
- **资金流**：充值存管 → 冻结 → 分账 → 提现，镜像银行子账户。
- **发票流**：业务交易确认单 → 6% 数电专票 →（B 线）进项发票台账。

## 硬校验：发生时就合规

四流不是事后审计，而是由代码在业务发生时**强制校验**：

> 无签署工单不可结算 · 无交付物不可验收 · 无验收不可开票 · 无发票依据不可分账

这些规则没有后门，任何一笔结算都必须四流齐备。

## 凭证归档与存证

每笔任务的合同、业务、资金、发票四类凭证统一归档，并写入**区块链存证哈希**。如对裁决或认定有异议，平台可一键导出**四流证据包**用于线下举证。

## 主动风控

平台内置多项风控规则实时预警：**用工集中度**监控、**防员工转零工**（历史发薪名单命中预警）、**反洗钱**（大额 / 高频 / 夜间）、**同 IP 多账号**关联检出。风险在发生时被看见，而不是事后追溯。
