广西零工云 · 系统详解

合规优先的灵活用工平台 —— 角色、流程、分支、字典全量拆解

广西零工云是立足广西的灵活用工平台，以「承揽后分包」模式运营：自身是企业与零工之间的承揽方，负责签约、付款、代扣代缴、开票与风控，并保留合同·业务·资金·发票「四流合一」的完整证据链。下文逐一列出全部角色与权限、类目工种、业务流程与分支、状态机、风控规则、后台任务、三方集成与配置项。

运营模式

承揽后分包：平台是承揽方，不是中介

企业把任务发包给平台（签《总承揽框架合同》+任务工单），平台再分包给零工（签《分包框架协议》+分包工单）。平台作为交易与责任主体，统一负责资金存管、代扣代缴、开票与合规留痕。

业务流程与分支

一共有哪些流程？每条分支怎么走

核心四流程：任务生命周期、结算引擎、税务引擎、争议处理；外加实名/绑卡/个体户登记/提现/充值/派单/投保/开票/申报等 9 条子流程。

① 任务全生命周期

1

发布

recruiting 招募中

企业发布，过发布风控（计酬方式+违禁词）

2

报名 / 派单

零工抢单(apply) 或 企业定向派单(dispatch)

3

录用

working 进行中

签分包工单 + 按单投保 + 冻结承揽价；防员转零校验

4

交付

delivered 待验收

零工上传交付物

5

验收·结算

settled 已结算

四流校验→算税→存管分账→开票

分支与出口

取消

cancelled：招募中任务可取消；解冻退回企业，通知报名者。

超期自动验收

交付后超 autoAcceptDays=14 天未验收 → 系统自动验收结算（保护零工）。

争议冻结

任务被设 dispute_id → 结算冻结，既不打款也不解冻，直至裁决执行。

限期重交付

争议裁 redeliver → 退回 working、清空交付物、冻结保留。

② 结算引擎（两阶段 + 四流硬校验）

阶段一：校验 + 建结算单（幂等屏障）。阶段二：四条「腿」依次执行，每条带幂等键；任一失败则整单保持 pending，由定时任务重试（≤8 次，超限转 failed 并告警）。

四流硬校验（任一不满足即阻断结算）

!
任务无争议锁（dispute_id 为空）
!

!

四条腿（幂等键 = 确认单号:腿名）

顺序腿资金流向 / 动作金额
1net存管转账：企业 → 零工零工实得 = 分包价 − 个税 − 增值税
2tax存管转账：企业 → 平台税款户（税+vat>0 时）代扣个税 + 代办增值税
3margin存管转账：企业 → 平台营收户（margin>0 时）平台毛利 = 承揽价 − 分包价
4invoice开数电票给企业（项目「*现代服务*灵活用工服务费」）承揽价全额（销项税率 6%）
资金分流（守恒） 承揽价(price，企业冻结并实付) = 分包价(gross) + 平台毛利(margin) 零工实得(net) = 分包价(gross) − 代扣个税(tax) − 代办增值税(vat) 平台税款户 ← tax + vat 平台营收户 ← margin 校验：net + tax + vat + margin = price ✓ 恒等 部分结算（争议）：按裁决金额结算，平台毛利按比例缩减，差额解冻退回企业

累计预扣公式（A线自然人）

A线自然人 · 累计预扣 本次应预扣 = max(0, 累计应纳税额 − 本连续段已预扣) 累计应纳税所得额 = 累计分包收入 × (1 − 20%) − 连续月数 × 5000元 累计应纳税额 = 七级累进速算(累计应纳税所得额) 连续段：同年内逐月回溯，遇无劳务报酬月份 即断，断后重新起算（2025年第16号公告） 增值税：月销售额 ≤ 10万元 免征； 超出部分按 1% 减按征收率代办

交付与验收

按工种结构化交付：填什么、传什么，因工种而异

平台连接劳务覆盖 16 类目 / 57 工种，而短视频剪辑、同城配送、家庭保洁、水电施工……每个工种「该交付什么、要核对什么」并不相同。交付环节因此不再是「一段自由说明 + 随意附件」，而是按工种动态生成的结构化表单：每个工种规定要填写的字段与要上传的材料，零工端动态渲染、企业端结构化验收、运营端在线可编辑模板。落地约 25 套差异化交付模板（22 个实体类目全覆盖 + 短视频剪辑 / 同城配送工种细化 + 默认兜底，仅「其他」类目走通用模板）。

① 三端协同：模板一处定义，三端各取所需

运营端 · 在线编辑

定义交付模板

按「大类基础模板 + 工种覆盖」维护每个工种的字段与上传项；存于系统配置，改即生效、自动审计。

零工端 · 动态表单

按工种填写上传

进交付页按所接工种拉取模板，动态渲染输入框 / 数字 / 日期 / 选择 / 分类上传；必填项不齐不可提交。

企业端 · 结构化验收

逐项核对验收

验收时按「字段 → 取值」清单 + 「按材料项分组」的可鉴权下载逐项核对，比一段文字更可核验。

模板解析与存储机制

  • 解析优先级：工种覆盖 byTrade > 大类基础 byCategory > 默认兜底 default
  • 快照式留痕：提交时把「字段 label/值、材料 label/文件」整体快照入库，运营日后改模板不污染历史交付
  • 四流校验不变：结构化内容自动生成交付物摘要，「无交付物不可验收」继续生效
  • 向后兼容：旧的「说明 + 附件」提交方式与历史数据照常工作
交付模板存于运营端可在线编辑的系统配置（同字典管理），无需发版即可调整某工种的填写/上传要求，变更自动写入审计日志。

全程留痕

证据链：四环节操作留痕 + 终端证据 + 防篡改 + 可导出

在「四流合一」合规凭证之外，平台把派单、抢单、单据管理、单据验收每一步操作同步沉淀为可追溯、可校验的操作证据链：谁、何时、从哪个 IP/设备、在什么现场位置做了什么，全部留痕；审计日志以哈希链固化防篡改，并可在企业端/运营端按单查看与一键导出，直接用于争议举证与税务核查。

① 每一步都留痕：终端证据自动捕获

操作人与时间

派单/抢单/录用/交付/验收等动作均记录操作人（角色）与精确时间，全量写操作审计无遗漏。

从哪 · 何设备

IP · 设备 · 现场定位

自动捕获发起方 IP 与设备 UA；零工报名/接单/交付经授权记录现场 GPS 定位，作为「本人在现场完成」的佐证。

防篡改

哈希链固化

每条审计记录以前后哈希串联成链，任意改、删、插入都会令其后记录对不上；运营端可一键校验链完整性并定位断点。

② 按单证据链：时间轴 + 四流凭证 + 一键导出

单笔工单从发布到结算的全流程操作按时间轴呈现，含环节、操作人、IP（企业端脱敏）、现场定位（可点开地图）。

四流凭证

合同哈希 · 交付物 SHA256

③ 安全与合规机制

个人信息保护

身份证号 AES-256-GCM 信封加密；手机号默认脱敏，完整 PII 需 user:read_pii 并审计；批量导出超 50 行需审批。

四流证据链

合同·业务·资金·发票全程留痕；争议可「四流归档一键导出」证据包。

资金存管不沾手

全程银行存管，充值由银行入账回调驱动；存管转账带幂等键，本地余额镜像银行子户。

权限纵深

RBAC 7 角色 + 25 权限点；敏感操作 TOTP step-up；登录失败锁定。

回调与上传安全

Webhook 按原始字节验签（验签失败告警）；上传校验文件头 magic number，非图片/PDF 强制附件下载。

资金应急开关

settlementPaused / withdrawalPaused 可一键暂停结算/提现（应急止血并告警）。
