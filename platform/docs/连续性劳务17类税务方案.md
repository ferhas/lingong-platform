# 连续性劳务（劳务报酬累计预扣）税务方案

> 面向"灵工云"平台的政策落地方案。回答两个问题：①政策依据与"17 类"到底指什么；②现有代码差距与改造方案。
> 编写日期：2026-06-13。政策为 2025 年 6 月发布、2025 年 10 月起全面执行，本方案按现行有效口径编写。

---

## 一、政策依据（来自税务总局公开文件）

平台从业人员的个税处理，核心是 2025 年这一组文件：

| 文件 | 内容 | 与本平台关系 |
|---|---|---|
| 国务院令第 810 号《互联网平台企业涉税信息报送规定》 | 平台须按季报送平台内经营者与从业人员的身份、收入信息 | 平台已有"涉税信息报送"模块（admin/tax/platform-report） |
| 国家税务总局公告 **2025 年第 15 号** | 涉税信息报送的具体口径（八类平台，含**灵活用工平台**） | 报送字段口径 |
| 国家税务总局公告 **2025 年第 16 号**（2025-06-26） | 平台为从业人员办理**扣缴申报 / 代办申报**：劳务报酬所得改按**累计预扣法**预扣预缴 | **本方案核心**，决定 A 线自然人怎么算个税 |

### 1.1 收入定性（劳务报酬 vs 经营所得）

税务总局对十三届人大第 8765 号建议的答复明确：平台从业人员收入按**经济实质**判定为两类——

- **劳务报酬所得**：从事设计、咨询、讲学、录音、录像、演出、表演、广告等劳务取得的收入，平台为**法定扣缴义务人**，按次/按累计预扣预缴个税。→ 本平台 **A 线（自然人）**。
- **经营所得**：注册为个体工商户，或虽未注册但实质从事生产经营活动的，收入属于经营所得，平台**不代扣**。→ 本平台 **B 线（个体工商户）**。

平台现有 A 线 / B 线双线模型与这一定性**完全吻合**，无需调整定性逻辑。

### 1.2 "17 类"是什么

"17 类连续性劳务"不是某份文件里的封闭编号清单，而是行业（云账户、好活等灵工平台）对**《个人所得税法实施条例》第六条所列劳务报酬项目**的归并叫法。实施条例原文列举的劳务项目为：

> 设计、装潢、安装、制图、化验、测试、医疗、法律、会计、咨询、讲学、新闻、广播、翻译、审稿、书画、雕刻、影视、录音、录像、演出、表演、广告、展览、技术服务、介绍服务、经纪服务、代办服务，以及其他劳务。

灵工平台通常把这些归并为约 17 个大类（设计/咨询/翻译/技术服务/演出表演/广告/录音录像……）作为可走"连续性劳务累计预扣"的**业务类目白名单**。

> 关键点：决定能否享受累计预扣的，**不是类目本身**，而是 ①收入定性为劳务报酬（A 线自然人），②在同一平台**按月连续**取得报酬。类目白名单的作用是业务/合规分类与报送，不是计税开关。

### 1.3 连续性劳务累计预扣法（16 号公告核心算法）

本期应预扣预缴税额 =

```
（累计收入 − 累计费用 − 累计免税收入 − 累计减除费用 − 累计其他扣除）× 预扣率 − 速算扣除数 − 累计减免税额 − 累计已预扣预缴税额
```

- **累计费用** = 累计收入 × 20%
- **累计减除费用** = 5000 元/月 × **当年截至本月在本平台连续取得劳务报酬的月份数**
- **预扣率表** = 综合所得年度七级超额累进（3% / 10% / 20% / 25% / 30% / 35% / 45%，速扣 0 / 2520 / 16920 / 31920 / 52920 / 85920 / 181920）
- **"连续"的中断**：某月在本平台**无报酬收入**则连续计数中断，之后重新起算（官方案例：8、9 月无收入，10 月起重新累计）。

与旧"按次预扣"（每次超 800 按 20%-40% 三级加成）相比，累计预扣对全年多笔小额收入**大幅降低预缴税负**（官方案例同样收入由全年预缴 5712 元降至 106.8 元）。

## 二、现有代码现状（已对照核对）

**结论：计税算法已经是 16 号公告的累计预扣法，方法论正确，但缺三块"政策外壳"。**

`platform/server/src/services/taxEngine.js#calcWithholding` 现状：

- ✅ 七级年度累进税率表 `BRACKETS` 与 16 号公告一致。
- ✅ `taxable = 累计收入 × (1 − laborExpenseRate) − months × monthlyDeduction`，其中 `laborExpenseRate=0.2`、`monthlyDeduction=5000`，即"累计收入×80% − 5000×月份数"，与公式一致。
- ✅ `tax = cumTax − prev.tax`（减累计已预扣），增值税月免征额监控也在。
- ✅ 税率/免征额/费用率全部走 `system_configs`，运营端可在线调。

**差距（gap）**：

1. **"连续月份数"未实现"断月重置"**。现 `months = COUNT(DISTINCT period) (+本月)`，是"当年有记录的月份数"，**不会因中间断月而重新起算**。与 16 号公告"断月重新累计"存在偏差——对断断续续接单的零工会**多扣减除费用、少缴税**，属合规风险点。
2. **没有"连续性劳务类目白名单"**。任务类目（设计/技术/翻译…）与政策无任何关联，缺少"本类目属于劳务报酬连续性劳务"的标记，报送与申报无法据此分类。
3. **扣缴申报/凭证未体现政策口径**。`/worker/tax/voucher`、`/admin/tax/declare-file` 未标注"累计预扣法/连续月份数/所得项目=劳务报酬"，税局申报底稿字段不全。

## 三、改造方案（建议分两期）

### P0（合规必做）——"连续月份"按政策口径计算

把"连续取得报酬月份数"改为**从本月往前回溯连续有收入的月份数**，断月即归零重算。

- 改动点：`taxEngine.js` 新增 `consecutiveMonths(workerId, period)`，从当前 period 往前逐月查 `tax_records` 是否有 `gross>0` 的 `cumulative` 记录，遇到空月停止；`calcWithholding` 的 `months` 改用它。
- 同时 `cumGross / prev.tax` 的累计区间应与"连续段"对齐（只累计本连续段内的月份），否则减除费用与累计收入口径不一致。建议把 `qYearStats` 改为"按连续段累计"，或新增 `period >= 段起点` 的过滤。
- 配套：`migration.test.mjs` 不受影响；在 `taxengine.test.mjs` 增加"断月重置"用例（1、2 月有收入→3 月断→4 月起 months 重新从 1 算）。

> 这是唯一涉及"算错税"的点，**优先级最高**。

### P1（政策外壳）——连续性劳务类目白名单 + 申报口径

1. **配置项**：`system_configs` 增 `continuousLaborCategories`（默认取实施条例归并的 17 类），运营端可维护。
2. **任务/报送打标**：发布时若 `category ∈ continuousLaborCategories` 且零工为 A 线自然人，则该笔 `tax_records` 标记 `income_type='labor_continuous'`；B 线个体户标记 `business`。
3. **申报与凭证**：`/worker/tax/voucher` 与 `/admin/tax/declare-file` 输出列增加"所得项目（劳务报酬）/ 计税方式（累计预扣）/ 连续月份数"，对齐税局扣缴端导入模板。
4. **报送对齐 15 号公告**：平台季度报送文件按"劳务报酬 / 经营所得"分类汇总。

### P2（体验，可选）

- 零工"收入与税务"页展示"连续接单 X 个月、本月已享减除费用 ¥XXXX、断月将重新起算"的说明，降低理解成本（与现有 `taxSummary.months` 字段衔接）。

## 四、风险与边界提示

- **减除费用同月不重复**：5000 元/月减除费用与工资薪金、其他平台的劳务报酬**共享**当月额度，平台只就本平台收入预扣，不代为合并扣除（年度汇算由个人完成）。文案需说明"按本平台收入预扣，非最终税负"。
- **VAT 与个税分轨**：增值税月免征额（现 10 万）逻辑独立于个税累计预扣，本方案不动。
- **B 线不受影响**：个体户走经营所得、平台不代扣，连续性劳务累计预扣**仅对 A 线自然人**生效。
- 本方案不改变四流硬校验、结算资金链与派单逻辑。

## 五、落地清单（已实现，2026-06-13）

- [x] **P0**：`taxEngine.js#consecutiveRun` 连续月份断月重置（同年内逐月回溯、断月即止）+ 累计区间对齐到连续段（`qSegStats`）+ 单测（断月重置 / 连续递增）
- [x] **P1**：`continuousLaborCategories` 配置（17 类白名单，运营端可改）+ 结算时按"类目白名单 + 主体类型"打标 `tax_records.income_type`（labor_continuous / labor_other / business）与 `consecutive_months`
- [x] **P1**：扣缴申报底稿 `GET /admin/tax/declare-file` 增列（所得项目 / 计税方式=累计预扣法 / 连续取得月份数）；零工年度凭证 `GET /worker/tax/voucher` 标注劳务报酬 + 累计预扣法 + 连续性劳务
- [x] **P1**：季度报送按所得类型分类汇总 `GET /admin/tax/quarter-summary?quarter=2026Q2`（对齐 15 号公告，labor_continuous / labor_other / business 三类人数·收入·税额）
- [x] **P2**：零工端 `GET /worker/income` 返回 `consecutiveMonths` / `cumulativeDeduction` / `taxNote`，小程序"收入与税务"页展示连续接单月份、已累计减除费用与断月重置说明

> 实现说明：A 线计税公式本就符合 16 号公告，本轮改造重点是 P0 的"断月重置"（修正断续接单会多扣减除费用的口径偏差）与 P1 的政策分类/申报口径外壳。B 线（经营所得）逻辑不变。
