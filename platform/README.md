# 灵工云 · 灵活用工平台（承揽后分包模式）

四端独立部署的生产级架构：

| 端 | 目录 | 技术栈 | 端口 |
|---|---|---|---|
| API端（后端服务） | `server/` | Node.js 22 + Express + SQLite(WAL) + JWT | 3000 |
| 企业端（B端） | `web-company/` | Vue 3 + Vite + Element Plus + Pinia | 5173 |
| 平台运营端（Admin） | `web-admin/` | Vue 3 + Vite + Element Plus + Pinia | 5174 |
| 零工端（C端） | `miniprogram-worker/` | 微信小程序（原生） | — |

API 契约见 [API.md](./API.md)。

## 业务闭环（承揽后分包）

1. **企业注册** → 行业负面清单自动评级 → **企业入驻审核**（通过即电子签《总承揽框架合同》）
2. **企业充值** 银行存管虚拟户 → **发布任务**（计酬方式白名单 + 伪劳务违禁词风控 → 冻结预算 → 生成《任务工单》）
3. **零工实名**（公安核验 + 银行卡三要素 → 电子签《分包协议（框架）》）→ **报名** → 企业**录用**（生成《分包工单》+ 按单投保）
4. **零工交付** → 企业**验收**（四流硬校验：无工单不可结算/无交付物不可验收）→ 生成《业务交易确认单》
5. **自动结算**：算税引擎按主体路由（自然人累计预扣个税 + 增值税10万免征额监控 / 个体户经营所得不代扣）→ 存管行分账（实发→零工、税费→备付金户、服务费→平台收益户）→ **全额开具6%数电发票**推送企业
6. **运营端**：风控预警（集中度/个体户引导/强制登记锁定）、月度批量申报、季度涉税报送、凭证归档（合同/业务/资金/发票四流凭证 + 区块链存证哈希）、**自动对账**（银行存管回执 ↔ 平台账务）
7. **B线通道**：零工在小程序内完成个体工商户登记后转入经营所得线（结算不代扣、解除接单锁定），平台获得进项改善税负

## 生产部署要点

- `server/.env`：必须设置强随机 `JWT_SECRET`、修改默认管理员密码、按需配置 `CORS_ORIGINS`/`DB_PATH`
- 金额内部以"分"存储；所有资金操作在 SQLite 事务内完成
- `src/integrations/` 为外部服务适配层（公安实名/存管银行/数电票/电子签/税务局/保险），生产替换为真实服务商凭据与SDK调用，接口签名不变
- Web 端 `npm run build` 产物用 Nginx 托管并反代 `/api` 到后端
- 数据库定期备份（SQLite 文件 + WAL）；并发量上升后可平滑迁移 PostgreSQL（schema 兼容）
- **前端安全响应头（Nginx，强烈建议）**：构建产物已注入基础 CSP `<meta>`（`script-src 'self'`），但 `frame-ancestors`/HSTS 等只能由响应头下发，请在 Nginx 站点补充（并用 `npm run preview` 在浏览器确认无 CSP 违规）：
  ```nginx
  add_header Content-Security-Policy "default-src 'self'; script-src 'self'; style-src 'self' 'unsafe-inline'; img-src 'self' data: blob:; connect-src 'self'; font-src 'self' data:; object-src 'none'; base-uri 'self'; frame-ancestors 'none'" always;
  add_header Strict-Transport-Security "max-age=31536000; includeSubDomains" always;
  add_header X-Frame-Options "DENY" always;
  add_header X-Content-Type-Options "nosniff" always;
  add_header Referrer-Policy "strict-origin-when-cross-origin" always;
  ```
- **令牌存储**：两 Web 端 access/refresh 令牌存于 `localStorage`（与小程序 Storage 一致）；XSS 面已通过"零 v-html/innerHTML/eval + 纯 JSON 接口 + 上述 CSP"显著收敛。更强方案为迁移 `httpOnly+Secure+SameSite` Cookie + CSRF（较大改造且与小程序模型不兼容，列为演进项）；`REFRESH_TOKEN_DAYS` 可缩短刷新令牌生命周期。

## v2 生产化能力(本轮新增)

### 权限体系
- **运营端 RBAC**:7 个预置角色(超级管理员 `*` / 审核专员 / 风控专员 / 财务税务 / 只读审计 / 客服 / 合规专员),权限点形如 `company:review`、`tax:declare`,控制菜单、按钮与 API(无权 403)。超管可在「用户管理」建号、分配角色、停用、重置密码。**合规专员**专司 PIPL 导出审批(`export:approve`)与完整 PII 查看(`user:read_pii`),与导出申请方职责分离;争议裁决(`dispute:rule`,动用冻结资金)由**风控专员**承担,一线客服仅可查看争议(最小权限)。
- **企业端子账号**:owner(主账号,全权)/ operator(发布/录用/验收)/ finance(充值/资金视角)。owner 在「成员管理」邀请成员(生成临时密码线下转交)。

### 安全与运维
- helmet 安全头、全局与敏感接口限流、登录连续失败5次锁定15分钟、密码策略(≥10位含字母数字)、生产环境强制 JWT_SECRET、pino 结构化日志 + request-id、全量写操作审计日志(运营端可查)。
- 业务参数(税率/阈值/类目/违禁词/负面清单/保费)入库,运营端「业务参数配置」在线编辑,算税/风控引擎实时生效。
- 输入侧增加疑似乱码拦截，避免姓名、企业名称、审核意见、回调备注等关键字段继续写入不可读文本。
- 文件上传(本地磁盘 + OSS 就绪适配层):交付物附件、B线进项发票;UUID 防遍历、mime 白名单、≤10MB、鉴权下载。
- 站内通知:审核结果/录用/交付/驳回/结算到账/风控锁定等关键事件,三端均有消息中心。

### 业务漏洞修复
- 任务取消(报名中)→ 冻结资金解冻;过期任务自动停止报名;零工可取消报名。
- B线个体户结算前**发票硬校验**(无进项发票不可结算,补全四流发票流)。

## v3 资金正确性与体验补全(本轮新增)

### 资金链路硬化(R1)
- **两阶段结算**:验收先落结算单据(幂等屏障)→ 逐腿分账(幂等键消重)→ 本地事务闭环;银行通道异常时单据停留 pending,**定时 Job 自动重试**,3次失败转 failed+高风险预警+通知财务。运营端「结算/提现单据」页全程监控。
- **提现申请单状态机**:申请冻结 → Job T+1 出金 → 到账划扣 / 失败自动解冻退回,全程通知;运营端可查全平台提现单。
- **任务超时治理**:交付 7 天未验收提醒企业、14 天自动验收结算(零工资金保护,天数可配置);逾期未交付/长期无人接单每日提醒。
- **T+1 按日对账**:每日凌晨自动比对银行回执与平台账务,差异落 `reconciliation_daily` 并产生预警,运营端按日明细可查。
- 6 个定时任务(node-cron):结算重试/提现处理/自动验收/超时提醒/按日对账/数据治理(通知清理、审计归档、孤儿文件删除);`scripts/backup.mjs` 数据库快照备份(保留14份)。
- 文件下载归属鉴权、临时密码 crypto 强随机、生产 CORS 白名单强校验、录用并发竞态消除。

### 运营完整性(R2)
- **JWT 刷新**:accessToken 2h + refreshToken 7d(轮换吊销),三端 401 透明刷新重放;登出吊销全部令牌。
- 企业/零工**详情抽屉**(全貌:账户/任务/成员/收入/合同/预警);风控预警带涉事对象**一键跳转**;**B端业务真实性证明包**一键导出;反洗钱基础规则(大额/高频/夜间);配置修改二次确认(旧值→新值对比);成员改角色;发布类目动态读配置。

### C端体验(R3)
- **微信一键登录**(开发态模拟/生产配 WX_APPID+WX_SECRET 即切官方接口),首登补手机号注册绑定;tabBar 未读红点;任务大厅排序(最新/报酬/报名少)+报酬区间筛选;近6个月收入柱状图;提现记录与状态;未实名引导直跳认证。

## v5 商用化完备(本轮新增)

### 资金真实闭环
- **充值单收银台**:企业创建充值单获取专属存管入金账户 → 对公转账 → **银行入金回调驱动入账**(本地余额是银行子账户镜像;金额不符自动挂起预警);充值单超时自动过期。开发态保留模拟入金与旧一步充值便于联调。
- **绑提现卡**:银行卡四要素核验 + 存管绑卡协议号,**出金凭协议号执行、平台不存卡号明文**;实名通过即自动开立存管子账户。
- **提现安全**:可配置强制短信验证码(`withdrawSmsRequired`);全局**结算/提现应急开关**(运营端一键止血,产生高风险预警);failed 结算单**人工重推**(flow:write + step-up)。
- **对账升级**:T+1 总额轧差升级为**逐笔核销**,差异自动落 `recon_diffs` 明细(bank_only/platform_only),运营端"对账差异工作台"逐笔处置留痕。

### 三方集成基础设施
- **统一回调入口** `/api/v1/webhooks/:provider`:原始报文 HMAC 验签(伪造回调自动预警)、事件幂等消重(`webhook_events`)、处理失败由**补单 Job 每5分钟自动重放**、运营端事件监控+手工重放。
- **出站调用日志** `integration_calls`(服务商/动作/时延/错误,运营端可查);健康检查覆盖 8 个适配器(实名/存管/数电票/电子签/税务/保险/短信/微信订阅)。
- 实名升级**三段式**(二要素→人脸核身→回传结果,`faceVerifyRequired` 开关);电子签新增**静默签授权**(企业一次性授权后工单自动落章);数电票支持**红字发票**(红冲留痕+关联重开)。

### 争议仲裁(平台介入)
- 状态机:协商期(48h,可在线和解撤回)→ 平台仲裁(举证72h)→ 裁决(全额/部分/不予结算/限期重交付)→ 执行 → 归档;超时自动流转(Job)。
- **争议锁**:争议创建即冻结该任务结算(既不解冻也不分账);**部分结算**按裁决金额计酬算税、企业按比例承担、剩余解冻退回、发票按实际承担额开具;已结算任务争议仅记录性裁决(不逆向扣划已提现资金)。
- 全程时间线留痕(举证/留言/裁决/执行),不服裁决可声明线下升级(平台备好四流证据包)。

### 客服工单与投诉
- 自建轻量工单(可挂任务/提现/发票/争议单据):提现/结算类自动**紧急**(SLA 2h 首响)、投诉举报自动高优并**联动风控预警**;SLA 超时自动升级提醒;办结后满意度评价;小程序建议同时接入微信客服(wx.openCustomerServiceChat)。

### 多通道消息触达
- 站内信 + **模板短信**(白名单场景:验证码/录用/驳回/结算到账/提现结果/账号冻结/争议进展)+ 订阅消息占位;模板运营端在线编辑可启停;外发日志含触达率统计(计费审计)。
- **短信验证码**服务:注册/登录/提现/绑卡/改密,频控防轰炸(60s/次、10条/天、5次错废码);开发态回显 devCode 便于联调。

### 安全与 PIPL
- **运营端 2FA(TOTP)**:自助绑定(otpauth)、登录二段验证、敏感操作 **step-up 复验**(裁决执行/导出审批/结算重推/应急开关/凭据创建须携带 X-TOTP-Code)。
- **身份证号信封加密**(AES-256-GCM + HMAC 查重索引,同证二绑拦截),仅投保/涉税报送场景解密且强制审计;`SECRETS_KEY` 独立密钥(生产建议 KMS)。
- **PII 最小可用**:零工手机号默认脱敏,完整查看须 `user:read_pii` 专项权限+审计;批量导出走**导出审批流**(申请→他人审批→48h限时下载→水印列)。
- **PIPL 落地**:实名前敏感信息**单独同意弹窗**(consent_idcard/face/bankcard 留痕);协议升版后**强制重新同意**;帮助中心含注销与个人信息删除指引。

### 财务功能中心

## v6 派单（定向派单，本轮新增）

任务到人在原「报名→录用」之外新增「派单→接受」一条路径,二者互补、共用同一套合规落地:

- **企业定向派单**:企业在任务管理「派单」选择候选零工(曾合作 ∪ 经本企业邀请码注册、已实名未锁定),发起带留言的派单邀约;任务详情可见派单记录与状态。
- **零工接受/拒绝**:小程序「我的接单」顶部展示派单邀约(含**收入预估**),零工接受后平台与其电子签《分包工单》+按单投保、任务进入进行中;拒绝则任务保持报名中,企业可改派或开放报名。
- **不破坏四流硬校验**:派单只是邀约,**零工接受时才签分包工单**——企业不能单方把任务绑定到零工身上,承揽关系仍以零工电子签为准;防员转零校验照常在录用落地时触发。
- 录用核心逻辑抽取为 `services/hiring.js#hireWorker`,报名录用与派单接受复用,落地完全一致;新表 `dispatches`、`applications.source` 列。

## v6 连续性劳务（16/15号公告，本轮新增）

A 线自然人劳务报酬本就按累计预扣法计税,本轮按国税总局 2025 年第 16 号公告补齐口径、按 15 号公告分类报送。详见 `docs/连续性劳务17类税务方案.md`。

- **断月重置（P0，影响实际税额）**:连续取得报酬月份数改为"同一纳税年度内从本月逐月向前回溯、遇无 A 线收入月份即中断",累计收入与累计已预扣均限当前连续段(`taxEngine.js#consecutiveRun`)。修正了断续接单多扣减除费用的口径偏差。
- **所得分类与打标（P1）**:新配置 `continuousLaborCategories`(连续性劳务类目白名单,约17类,运营端可改);`tax_records`/`settlements` 增 `income_type`(连续性劳务报酬/其他劳务报酬/经营所得)与 `consecutive_months`,结算时按"类目白名单+主体类型"自动判定。
- **申报与报送（P1）**:扣缴申报底稿增列(所得项目/计税方式=累计预扣法/连续取得月份数);零工年度凭证标注;新增 `GET /admin/tax/quarter-summary` 季度按所得类型分类汇总,运营端「税务工作台」季度报送区**可视化展示**连续性劳务/其他劳务/经营所得三类的人数·收入·税额。
- **零工端展示（P2）**:小程序「收入与税务」展示连续接单月份数、已累计减除费用(5000×月份)与断月重置说明。
